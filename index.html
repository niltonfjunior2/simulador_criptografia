Como funciona a matemática da Cifragem?

O simulador converte cada caractere de forma independente usando o princípio estrutural do algoritmo RSA. O processo ocorre em três etapas fundamentais:

  • Mapeamento Numérico (ASCII): O sistema captura o caractere e usa a função interna charCodeAt(0) para extrair o código numérico real da tabela ASCII (por exemplo, a letra 'A' é o número 65, o símbolo '!' é o número 33).
  • Exponenciação da Chave: O código numérico original (chamado de M) é elevado à potência do expoente público do sistema, definido neste ambiente como e = 7. Matematicamente, calcula-se M⁷.
  • Aritmética Modular (Engrenagem Numérica): O resultado da potência passa pela operação de módulo (resto da divisão inteira) utilizando o número base N = 253. A fórmula final executada pelo Remetente é: C = (M⁷) mod 253. O valor resultante C é o número cifrado que pode trafegar de forma segura.

Como funciona a matemática da Decifragem?

A decifragem prova o conceito de função de mão única com alçapão. Para reverter o cálculo modular anterior sem ter que testar números por força bruta, o Receptor usa a sua chave secreta como um atalho algébrico:

  • O Alçapão Inverso: O par de chaves em um sistema RSA obedece à regra estrita onde (e * d) mod φ(N) = 1. Para este ambiente didático operar com qualquer caractere padrão do teclado, os números primos de base escolhidos foram 11 e 23, gerando o módulo N = 253 e uma função Totiente de Euler φ(N) = 10 * 22 = 220. Como configuramos o expoente público como e = 7, a matemática nos obriga a usar o expoente privado complementar secreto d = 63, pois (7 * 63) = 441, e 441 mod 220 = 1.
  • Cálculo do Atalho: Quando o Receptor valida o acesso com a chave privada correta, o sistema pega cada número cifrado recebido (C) e o eleva à potência secreta de 63 (C⁶³).
  • Restauração Modular: Ao aplicar o resto da divisão pelo módulo (mod 253), as propriedades dos teoremas aritméticos garantem a restauração exata do código ASCII original: M = (C⁶³) mod 253. O JavaScript então traduz esse número de volta na letra legível usando String.fromCharCode(m).
  • O comportamento do hacker: Se você tentar usar a chave pública para descriptografar, o script rodará a fórmula simulando o expoente errado (usando 7 de novo em vez de 63). Como 7 não é o inverso multiplicativo de si mesmo sob o módulo 220, a equação quebra e gera apenas símbolos corrompidos.
